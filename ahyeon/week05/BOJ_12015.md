### 📖 풀이한 문제

- 백준 12015 - 가장 긴 증가하는 부분 수열 2
- 문제 링크: https://www.acmicpc.net/problem/12015
- 난이도: gold2
- 걸린 시간: 30m

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어: dp
- 고려한 시간복잡도: O(n^2) -> O(nlogn)
- 막힌 부분 / 해결 방법:
  - 이 문제의 선수 문제인 [가장 긴 증가하는 부분 수열](https://www.acmicpc.net/problem/11053) 이랑 똑같이 풀려고 했다. 문제는 같은데 n의 크키가 달랐다(1,000 vs 1,000,000)
  - 완전탐색식 dp로 풀면 O(n^2)이기 때문에 시간초과가 났다. 이 방식은 i에서 끝나는 lis의 길이를 구하는 거고, 점화식이 보이는 쉬운 dp 문제였다.
  - 이번에 사용한 방법은 tails 배열을 사용하는 문제였다. tails[k] 는 길이가 k+1짜리 lis의 최소 끝 값이다. 점화식으로 푸는 dp 문제랑은 다르게 그리디와 자료구조를 이용하는 풀이방식이다
  - tails는 가능한 최소 끝 값만 유지한다. 새로운 값이 들어갈 위치를 이진탐색을 통해 찾고, 해당 값이 현재 배열의 길이보다 크면(최대값보다 크면) 배열의 뒤에 붙여 배열의 크기를 늘리고, 그렇지 않으면 해당 위치를 교체한다.
  - dp는 모든 경우 확인, 현재 방식은 최적 상태만 유지

### ⭐️ 문제에서 주로 사용한 알고리즘

- 이분탐색

### 🧠 대략적인 코드 설명

- python

```
import sys
input = sys.stdin.readline

from bisect import bisect_left

n = int(input())
nums = list(map(int, input().split(" ")))

tails = []

for num in nums:
    pos = bisect_left(tails, num)

    if pos == len(tails):
        tails.append(num)
    else:
        tails[pos] = num

print(len(tails))
```
