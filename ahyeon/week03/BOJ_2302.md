### 📖 풀이한 문제

- 백준 2302 - 극장 좌석
- 문제 링크: https://www.acmicpc.net/problem/2302
- 난이도: gold5
- 걸린 시간: 20m

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어: 전형적인 dp(피보나치) + 구간 나누기
- 고려한 시간복잡도: O(n)
- 막힌 부분 / 해결 방법:

### ⭐️ 문제에서 주로 사용한 알고리즘

- dp

### dp는 어떻게 풀어야 할까

#### 사고 순서

1. 상태 정의(state)
   먼저 dp[i] 가 무엇인지 정의한다.
   이 문제에서는 dp[i]가 좌석 i개일 때 가능한 경우의 수이다.

2. 마지막 선택 기준으로 분해
   dp는 거의 항상 '마지막 행동 기준' 또는 '첫 행동 기준'으로 나눈다.
   이 문제의 경우 1번이 그대로 앉는다 or 1번과 2번이 교환해 앉는다 이렇게 둘로 나뉜다.
   그래서 dp[n] = dp[n-1] + dp[n-2]

3. 초기값
   점화식만 있으면 안된다.
   dp[0], dp[1] 같은 base case가 필요하다.
   이 문제의 경우
   dp[0] = 1, dp[1] = 1, dp[2] = 2

4. 계산 순서
   dp는 보통 작은 문제에서 큰 문제로 확장해 나간다.

#### dp의 5가지 유형

대부분 dp문제는 아래 유형 중 하나이다.

1. 피보나치 형
   `dp[n] = dp[n-1] + dp[n-2]`

예) 극장 좌석, 계단, 타일링 같은 문제들

2. 선택 최대값
   `dp[n] = max(...)`

예) 스티커, 카드구매,

3. 구간 dp
   `dp[i][j]`

예) 행렬 곱, 펠린드롬

4. 배낭(knapsack)
   `dp[i][w]`

예) 물건 선택

5. LIS(증가 부분 수열)
   `dp[i] = ...`

#### dp 문제인지 판단하는 법

문제 읽다가 이 단어 나오면 바로 dp인지 의심

1. 경우의 수
2. 최대/최소
3. 선택
4. n<=1000 (브루트포스는 보통 불가능)

### 🧠 대략적인 코드 설명

- python

```
import sys
input = sys.stdin.readline

n = int(input().rstrip())
m = int(input().rstrip())
vip = [int(input()) for _ in range(m)]

dp = [0] * 41
dp[0] = 1
dp[1] = 1
dp[2] = 2

for i in range(3, 41):
    dp[i] = dp[i-1] + dp[i-2]

answer = 1
prev = 0

for v in vip:
    length = v - prev - 1
    answer *= dp[length]
    prev = v

# 마지막 구간
length = n - prev
answer *= dp[length]

print(answer)
```
