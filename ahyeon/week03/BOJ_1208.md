### 📖 풀이한 문제

- 백준 1208 - 부분 수열의 합 2
- 문제 링크: https://www.acmicpc.net/problem/1208
- 난이도: gold 1
- 걸린 시간: 30m

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어: 재귀 탐색, 완탐
- 고려한 시간복잡도: O(2^n)
- 막힌 부분 / 해결 방법:
  - 부분 수열의 합(1182)와 같은 문제라서 같은 방식으로 풀어보았는데 당연하게도 틀렸다 ㅋ 시간초과
  - 1182번은 `n<=20`이라서 `2^20 = 약 100만`, dfs가 가능했다.
  - 이 문제는 `n<=40`이라서 `2^40 = 약 1조`, 불가능 시간초과 난다.
  - 따라서 Meet in the Middle(MITM) 알고리즘을 사용했다.
  - 이 문제는 수열의 합이 s가 되는 수열의 개수를 구하는 문제인데, 절반으로 나눠서 각 조합의 합을 계산해두고, 이후에 왼쪽 오른쪽 합들을 조합해서 s가 되는 걸 찾으면 된다.
  - sort()는 O(n log n)
  - bisect_left, bisect_right 는 O(log n)
  - 결과적으로 내 코드의 시간복잡도는 2^(n/2) log(2^(n/2))

### ⭐️ 문제에서 주로 사용한 알고리즘

- MITM, 재귀탐색

### 🧠 대략적인 코드 설명

- python

```
import sys
input = sys.stdin.readline
from bisect import bisect_left, bisect_right

def dfs(end, idx, sum, sumList):

    if idx == end:
        sumList.append(sum)
        return

    dfs(end, idx+1, sum, sumList)
    dfs(end, idx+1, sum+nums[idx], sumList)


n, s = map(int, input().split())
nums = list(map(int, input().split()))

sumA = []
sumB = []

dfs(n//2, 0, 0, sumA)
dfs(n, n//2, 0, sumB)

sumB.sort()

answer = 0

for a in sumA:
    target = s - a
    left = bisect_left(sumB, target)
    right = bisect_right(sumB, target)
    answer += (right - left)

if s == 0:
    answer -= 1

print(answer)
```
