### 📖 풀이한 문제

- 백준 1520 - 내리막 길
- 문제 링크: https://www.acmicpc.net/problem/1520
- 난이도: gold3
- 걸린 시간: 40m

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어: 그래프 탐색 문제라고 생각했다. 단순한 dfs, bfs 로 풀려고 했지만 격자 크기가 최애 500\*500이어서 모든 경로를 탐색하면 시간초과가 발생할 것이라 생각했다.
- 고려한 시간복잡도: O(n*m*4) = O(nm)
- 막힌 부분 / 해결 방법:
  - 동일한 칸을 여러 번 탐색하지 않도록 메모이제이션을 활용한 dfs + dp로 풀었다.
  - dp 상태정의를 어떻게 해야할 지 헷갈렸다. 처음에는 한 칸에 동서남북 접근 여부를 저장하는 방식을 생각했다. 하지만 이 문제는 항상 높이가 감소하는 방향으로만 이동하기 때문에 그래프에 사이클이 존재하지 않는 DAG구조가 된다. 그래서 방문 방향을 저장할 필요 없이 `dp[x][y] = 여기서 목적지까지 갈 수 있는 경로의 수` 만 저장하면 된다.
  - 보통 반복문이 많은 경우에 python3보다 pypy가 빠른 경우가 많아서 pypy를 애용하는데, 이 문제는 메모리초과가 났다. python3가 - c로 구현된 인터프리터 - 메모리 사용량이 비교적 안정적 - recursion이나 리스트 처리에서 메모리 효율 좋음
    이라는 특징이 있다면 pypy는 - JIT기반 python 인터프리터 - 실행중에 코드를 컴파일해서 최적화 - 반복문 많은 문제에서 속도 빠름 - 메모리 사용량 비교적 많음
    이라는 특징이 있다. 따라서 python3으로 제출했더니 잘 통과했다.

### ⭐️ 문제에서 주로 사용한 알고리즘

- dfs, 메모이제이션, dp

### 🧠 대략적인 코드 설명

- python

```
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**6)

m, n = map(int, input().split())
board = [list(map(int, input().split(" "))) for _ in range(m)]

dp = [[-1]*n for _ in range(m)]
dirs = [[1,0], [-1,0], [0,1], [0,-1]]

def dfs(x, y):
    if x == m-1 and y == n-1:
        return 1

    # 이미 계산된 경우
    if dp[x][y] != -1:
        return dp[x][y]

    dp[x][y] = 0
    for di, dj in dirs:
        ni = x + di
        nj = y + dj

        if 0<=ni<m and 0<=nj<n:
            if board[ni][nj] < board[x][y]:
                dp[x][y] += dfs(ni, nj)

    return dp[x][y]

print(dfs(0,0))
```
