### 📖 풀이한 문제

- 백준 1937 - 욕심쟁이 판다
- 문제 링크: https://www.acmicpc.net/problem/1937
- 난이도: gold3
- 걸린 시간: 30m

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어: 1520에서 내리막을 오르막으로 바꾸기
- 고려한 시간복잡도: O(n^2)
- 막힌 부분 / 해결 방법:
  - 1520 내리막길 문제를 직전에 풀었는데, 해당 문제는 처음과 끝이 명확하게 정해진 문제였다. 그리고 경로 개수를 묻는 문제였다. 하지만 이번 문제는 처음과 끝이 정해지지 않았고, 최대로 이동할 수 있는 길이를 묻는 문제였다.
  - 이번 문제에서 `dp[i][j] = (i, j)에서 시작했을 때 판다가 살 수 있는 최대 날짜` 이었기 때문에 점화식은 max로 최대를 구하는 것이고, 현재 저장되어 있는 날짜와 새롭게 구한 날짜 중 하나를 택하는 방식으로 점화식을 짰다.
  - 메모이제이션을 하면서 풀었기 때문에 각 칸 dfs는 1번씩만 계산하면 되고, 그렇기 때문에 최대 시간 복잡도도 500\*500 = 250000으로 통과가 가능하다.

### ⭐️ 문제에서 주로 사용한 알고리즘

- dp, dfs, 메모이제이션

### 🧠 대략적인 코드 설명

- python

```
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**6)

n = int(input())
forest = [list(map(int, input().split(" "))) for _ in range(n)]

dp = [[-1]*n for _ in range(n)]
dirs = [[0,1], [0,-1], [1,0], [-1,0]]

def dfs(i, j):
    if dp[i][j] != -1:
        return dp[i][j]

    dp[i][j] = 1

    for di, dj in dirs:
        ni = i + di
        nj = j + dj
        if 0<=ni<n and 0<=nj<n:
            if forest[ni][nj] > forest[i][j]:
                dp[i][j] = max(dp[i][j], dfs(ni, nj) + 1)

    return dp[i][j]

answer = 0

for i in range(n):
    for j in range(n):
        answer = max(answer, dfs(i, j))

print(answer)
```
