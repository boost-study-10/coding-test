### 📖 풀이한 문제

- 프로그래머스 258707 - n+1 카드게임
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/258707
- 난이도: lv3
- 걸린 시간: 45m

### 🧩 풀이 설계 / 막힌 부분

- 간단 문제 설명:
  - 처음에 n/3개의 카드를 갖는다. 그 이후 각 라운드마다 2장의 카드를 추가로 뽑게 되며 갖고싶으면 동전 1개씩을 낸다. 갖고싶지 않으면 동전을 쓰지 않고 카드는 버린다.
  - 카드에 적힌 수의 합이 n+1이 되는 카드 두장이 있으면 내고 다음 라운드로 간다. 카드를 낼 수 없으면 게임을 종료한다.
- 처음 접근 아이디어:
  - 그리디 알고리즘으로 접근했다. 카드를 뽑았을 때 되는 숫자가 있으면 카드를 머고 아니면 버린다.
- 고려한 시간복잡도:
- 막힌 부분 / 해결 방법:
  **막힌 부분**
  - 그리디로 풀다보면 방금 버린 카드를 바로 다음 턴이나 다다음 턴에 써야하는 경우가 생긴다.
  - 그리디로는 풀 수 없다는 것을 알았다.

  **해결 방법**
  - 버린 카드를 활용할 수 있어야한다.
  - 버린 카드를 바로 지우지 말고, 대기열을 만들어 뽑는 2장 카드들을 넣는 리스트를 넣어놓는다.
  - 이후에 동전을 사용하는 순서는 다음과 같다.
    1. 현재 가지고 있는 카드로 n+1을 만들 수 있으면 굳이 동전 쓰지 않고 통과
    2. 1이 안될 때 시도. 현재 가지고 있는 카드 1장, 대기열 카드 1장으로 n+1을 만들 수 잇으면 동전을 하나 써서 통과
    3. 2도 안될 때 시도. 대기열 카드 2장을 써서 n+1을 만들 수 있으면 동전 2개를 쓰고 통과
    4. 3까지 안되면 게임 종료

```
from collections import deque

def makeval(d1, d2, val):
    newd2 = set(d2)
    for card in d1:
        if val - card in newd2:
            d1.remove(card)
            d2.remove(val-card)
            return True
    return False

def solution(coin, cards):
    answer = 0
    n = len(cards)
    val = n+1
    q = deque(cards[n//3:])
    pending = []
    my = cards[:n//3]

    while coin >= 0 and q:
        pending.append(q.popleft())
        pending.append(q.popleft())

        if makeval(my, my, val):
            pass
        elif coin >= 1 and makeval(my, pending, val):
            coin -= 1
        elif coin >= 2 and makeval(pending, pending, val):
            coin -= 2
        else:
            break
        answer += 1

    return answer + 1
```
