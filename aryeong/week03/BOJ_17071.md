### 🧩 풀이 설계 / 막힌 부분
- 처음 접근 아이디어: 동생 위치를 계산하고 bfs를 사용하자.
- 고려한 시간복잡도:
- 막힌 부분 / 해결 방법: 동생은 매 초마다 k + 1 + 2 + ... + t 만큼 이동하므로 시간에 따라 위치가 계속 변한다. 동생의 위치 -> k + 1/2 * t(t+1)​. 매 시간 t마다 동생의 위치를 계산하면서
그 시간에 수빈이가 해당 위치에 도달할 수 있는지 확인해야 한다.

방문 배열을 visited[2][position] 형태로 만들어 visited[time % 2][pos]로 관리한다.
이렇게 하면 같은 위치라도 시간 짝/홀 에 따라 다른 상태로 처리할 수 있다.

### ⭐️ 문제에서 주로 사용한 알고리즘
- bfs

### 🧠 대략적인 코드 설명
```js
const input = require("fs")
  .readFileSync("/dev/stdin")
  .toString()
  .trim()
  .split("\n");

const [n, k] = input[0].split(' ').map(Number);
const MAX = 500000;

function go(x) {
    const visited = Array.from({ length: 2 }, () => Array(MAX + 1).fill(0));
    let queue = [x];
    visited[0][x] = 1;
    let time = 0;

    while (true) {
        const target = k + (time * (time + 1)) / 2;
        if (target > MAX) return -1;
        if (visited[time % 2][target]) return time;
        
        const nextQueue = [];
        for (let i = 0; i < queue.length; i++) {
            const current = queue[i];
            const moves = [current - 1, current + 1, current * 2];

            for (const next of moves) {
                if (next >= 0 && next <= MAX && !visited[(time + 1) % 2][next]) {
                    visited[(time + 1) % 2][next] = 1;
                    nextQueue.push(next);
                }
            }
        }

        queue = nextQueue;
        time++;
    }
}

console.log(go(n));
```
