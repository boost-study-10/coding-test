### 🧩 풀이 설계 / 막힌 부분
- 처음 접근 아이디어: 처음에 생각했던 건 O(n^2)이라 시간 복잡도 초과. 스택을 사용하라는 힌트 얻음. 1. 오큰수 못 찾은 인덱스를 스택에 추가 2. 현재 숫자가 스택의 top보다 크면 스택의 인덱스 -> 현재 숫자 3. 더 이상 작지 않을 때까지 계속 pop
- 고려한 시간복잡도: O(n)

### 🧠 대략적인 코드 설명
```js
const input = require("fs")
    .readFileSync("/dev/stdin")
    .toString()
    .trim()
    .split("\n");

const n = Number(input[0]);
const arr = input[1].split(' ');

const stack = [];
const ans = Array(n).fill(-1);

for (let i = 0; i < n; i++) {
    while(stack.length && arr[stack[stack.length - 1]] < arr[i]) {
        const idx = stack.pop();
        ans[idx] = arr[i];
    }
    stack.push(i);
}

console.log(ans.join(' '));
```
