### 📖 풀이한 문제

- 백준 9019 - DSLR
- 문제 링크: https://www.acmicpc.net/problem/9019
- 난이도: 골드4
- 걸린 시간: 60분 / 6트(메모리 초과/시간초과/런타임에러/틀렸습니다 모두 봤음........)

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어: 시작 숫자에서 목표 숫자까지 변환하는 최소 명령어 개수를 구해야 하므로 최단 거리 탐색..? BFS?
- 고려한 시간복잡도:
  - A, B 범위가 0~10,000일 때, 4가지 연산 수행 가능한 BFS하면 O(10000 \* 4)
- 막힌 부분 / 해결 방법
  1. visited 체크 안 함
  2. 변수명 수정 덜 함
  3. 경계값 조건 잘못 처리함

-> 답이 안나와도 당황하지 말고~ 천천히 다시 풀기 ㅎ.ㅎ

### ⭐️ 문제에서 주로 사용한 알고리즘

- BFS
  - 경로를 구하고있어서 DFS를 탐색하려고 했지만..
  - 구하는 것은 최소 연산 횟수이므로 최단 경로를 보장하는 BFS가 적합함

### 🧠 대략적인 코드 설명

1. 시작 숫자에서 BFS 진행
   - 4가지 연산과 명령어를 객체로 저장하고 순회하면서 실행
   - 도착 숫자라면 큐에 저장한 명령어 반환
