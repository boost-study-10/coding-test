### 📖 풀이한 문제

- 백준 2583 - 영역 구하기
- 문제 링크: https://www.acmicpc.net/problem/2583
- 난이도: 실버1
- 걸린 시간: 40분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어: 그래프 탐색 BFS/DFS
- 고려한 시간복잡도:
  - M = 100, N = 100, K = 100일 때, BFS/DFS 탐색 시 O(M \* N) = 10,000
- 막힌 부분 / 해결 방법:

- 지문에서 원점이 좌하단(0, 0)으로 시작하지만,
  일반적인 배열 그래프는 좌상단이 시작점이라 index 맞추는 과정이 오래걸림
  - 근데, 해당 문제는 영역의 개수와 크기를 구하면 되므로,
  - 좌표계 변환 없이 회전된 상태로 탐색해도 연결된 영역의 개수와 크기는 동일함
- 굳이 인덱스 변환없이 그래프 탐색 진행..

### ⭐️ 문제에서 주로 사용한 알고리즘

- BFS
  - 그래프 탐색 문제여서 DFS도 가능

### 🧠 대략적인 코드 설명

1. 입력받은 두 좌표에 포함되는 영역을 1로 설정
   - 두 좌표가 (a, b), (c, d)라면 영역은 (a~c, b~d)
2. 모든 영역의 좌표를 돌면서 0인 곳에서 탐색 진행
3. 탐색한 depth를 저장한 후 정렬
