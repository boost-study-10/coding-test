### 📖 풀이한 문제

- 백준 16234 - 인구 이동
- 문제 링크: https://www.acmicpc.net/problem/16234
- 난이도: 골드4
- 걸린 시간: 64분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어:
  - 모든 나라를 순회하면서 BFS를 이용해 연합 찾기
  - 인접한 두 나라의 인구 차이가 L 이상 R 이하라면 해당 나라 저장
  - 연합에 속한 나라들의 인구 평균을 구해서 한 번에 갱신
  - 하루가 끝났을 때 단 한 곳이라도 인구 이동이 발생했다면 날짜를 1 증가시키고,
  - 더 이상 연합이 생기지 않으면 종료
- 고려한 시간복잡도:
  - 하루 동안 모든 칸을 최대 한 번씩 방문하므로 O(N^2)
  - BFS 안에서도 각 나라가 그날 최대 한 번만 큐에 들어가므로 추가 비용도 O(N^2) 안에 포함됨
  - N의 최대가 50 일 때, O(Day × N^2)
- 막힌 부분 / 해결 방법
  - 방문하지 않은 모든 칸에서 BFS를 시작하면, 이동할 수 없는 단일 나라를 어떻게 처리할지 고민..
  - BFS 결과로 만들어진 연합의 크기를 확인
    - 연합 크기가 1이면(시작점만 포함) 인구 이동이 없는 경우
    - 연합 크기가 2 이상이면 인구 이동이 발생한 경우
  - BFS를 시작할 때 시작 나라를 바로 visited 처리하고, 연합 목록과 인구 합계에 즉시 포함시키는 것이 좋을듯

### ⭐️ 문제에서 주로 사용한 알고리즘

- BFS

### 🧠 대략적인 코드 설명

1. while로 매 날짜마다
2. visited 배열을 새로 만들고, 그날 인구 이동이 있었는지 확인할 flag를 false로 초기화
3. 모든 칸을 순회하면서 아직 방문하지 않은 나라라면 BFS를 수행
   - 인접한 나라와의 인구 차이가 L 이상 R 이하이면 같은 연합으로 묶고,
   - 좌표와 인구 합 저장
4. BFS가 끝난 뒤 연합 크기가 2 이상이면 평균 인구수를 계산하고, 연합에 속한 모든 나라의 인구를 그 값으로 갱신
5. 이 과정에서 하나라도 연합이 만들어졌다면 flag = true로 두고, 다음날 진행
6. 만약 하루 동안 단 한 번도 연합이 생기지 않았다면 종료
