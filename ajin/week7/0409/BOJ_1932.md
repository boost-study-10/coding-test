### 📖 풀이한 문제

- 백준 1932 - 정수 삼각형
- 문제 링크: https://www.acmicpc.net/problem/1932
- 난이도: 실버1
- 걸린 시간: 30분

### 🧩 풀이 설계 / 막힌 부분

- 문제: 삼각형의 맨 위에서 시작해서 아래층으로 내려갈 때, 바로 아래 또는 아래 오른쪽 칸으로만 이동할 수 있을 때 얻을 수 있는 최대 합 구하기
- 처음 접근 아이디어:
  - 각 칸에 도달하는 경로는 바로 위 왼쪽 칸 또는 바로 위 칸 두 곳 중 하나에서만 올 수 있으므로
    각 위치까지의 최대 합을 누적해서 구하기
- 고려한 시간복잡도: 삼각형의 모든 칸을 한 번씩 확인하므로 1 + 2 + ... + n번 연산해서 O(n^2)
- 막힌 부분 / 해결 방법: X

### ⭐️ 문제에서 주로 사용한 알고리즘

- DP

### 🧠 대략적인 코드 설명

1. 각 줄의 숫자를 입력받아 dp 배열에 저장하고, 뒤쪽은 길이를 맞추기 위해 0으로 채움
2. dp[i][j]를 i번째 줄의 j번째 칸까지 왔을 때 얻을 수 있는 최대 합으로 사용
3. 두 번째 줄부터 아래로 내려가면서
   - 가운데 칸들은 dp[i - 1][j - 1]와 dp[i - 1][j] 중 더 큰 값을 더해 갱신
   - 각 줄의 마지막 칸은 바로 위 오른쪽 값이 없으므로 dp[i - 1][i]를 더해 따로 처리
4. 마지막 줄에서 가장 큰 값을 출력

### 다른 풀이

- 삼각형 원본 값은 nums에 따로 저장하고, 최대 합 계산용 dp 배열을 별도로 만들기
- dp[i][j]를 i번째 줄 j번째 칸까지의 최대 합으로 두고,
  - 이전 줄의 왼쪽 위 칸과 바로 위 칸 중 가능한 위치의 최댓값을 선택해 현재 값을 더함
- 원본 데이터와 DP 테이블이 분리되어 있어서 로직을 명확하게 파악할 수 있음
