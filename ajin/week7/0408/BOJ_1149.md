### 📖 풀이한 문제

- 백준 1149 - RGB거리
- 문제 링크: https://www.acmicpc.net/problem/1149
- 난이도: 실버1
- 걸린 시간: 19분

### 🧩 풀이 설계 / 막힌 부분

- 문제: N개의 집을 빨강, 초록, 파랑 중 하나로 칠할 때, 인접한 집의 색이 같지 않도록 칠하는 최소 비용 구하기
- 처음 접근 아이디어:
  - 현재 집은 이전 집은 나머지 두 색 중 하나여야 하므로
  - DP를 사용해 각 집에 대해 모든 색을 칠한 경우를 구하기
- 고려한 시간복잡도:
  - 집의 개수 N <= 1000일 때, 각 집마다 3가지 색에 대해 이전 집의 2가지 경우만 비교하면 되므로 O(N)
- 막힌 부분 / 해결 방법: X

### ⭐️ 문제에서 주로 사용한 알고리즘

- DP

### 🧠 대략적인 코드 설명

1. colors에 이전 색과 현재 가능한 색을 인덱스로 저장
2. 두 번째 집부터 마지막 집까지 순회하면서
   현재 집의 색이 a라면, 이전 집은 b 또는 c 색만 가능하도록 최소 비용 구하기
   - cost[i][a] += Math.min(cost[i - 1][b], cost[i - 1][c])
3. 마지막 집의 세 색 중 최솟값이 전체 최소 비용 출력하기
