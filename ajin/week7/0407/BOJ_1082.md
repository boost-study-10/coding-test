### 📖 풀이한 문제

- 백준 1082 - 방 번호
- 문제 링크: https://www.acmicpc.net/problem/1082
- 난이도: 골드3
- 걸린 시간: 28분

### 🧩 풀이 설계 / 막힌 부분

- 문제: 숫자 0 ~ n-1에 대한 비용이 주어질 때, 가진 돈 M으로 가장 큰 방 번호 구하기
- 처음 접근 아이디어:
  - 숫자별 가격이 다양하고(배수 관계가 아님) 자릿수와 앞자리 숫자가 중요하므로 그리디는 ~불가능~한 줄 알았는데, 가능하다고한다..
  - 숫자를 0원부터 M원까지 최대한 구매했을 때 경우를 모두 구하면서 DP로 업데이트하기
- 고려한 시간복잡도:
  - M <= 50이고 숫자는 0~9까지 존재하므로
  - 각 숫자에 대해 가능한 금액 m을 순회하면 O(N \* M)
- 막힌 부분 / 해결 방법:
  - M이 50일 때, 50자리 숫자가 생길 수 있으므로 JS 숫자 연산은 BigInt로 처리하기

### ⭐️ 문제에서 주로 사용한 알고리즘

- DP
- (또는 그리디)

### 🧠 대략적인 코드 설명

1. dp를 돈으로 만들 수 있는 가장 큰 방 번호로 저장
2. 큰 숫자부터 하나씩 보면서 DP 갱신
3. 현재 숫자를 최대한 구매하기(m: P[num]원 ~ M원)
4. m - P[num]원을 써서 만든 기존 숫자 뒤에 새로운 숫자를 붙일 때(새로운 후보)와
   - dp[m - P[num]] \* 10n + num
5. 기존 dp[m]를 비교해 새로운 후보가 더 크면 갱신
6. dp[M] 출력

### 다른 풀이(그리디)

> 자릿수를 최대한 많이 확보하고, 그 다음 앞자리부터 가능한 한 큰 숫자로 교체하기

1. 가장 싼 숫자를 최대한 많이 사서 자릿수를 먼저 확보
2. 이후 가장 앞자리부터, 현재 숫자를 더 큰 숫자로 바꿀 수 있는지 확인
3. 교체에 필요한 추가 비용이 남은 돈 이하라면 더 큰 숫자로 바꾸기
4. 이를 앞자리부터 반복해 가장 큰 방 번호를 만들기
5. 단, 싼 숫자가 0인 경우 예외처리 필요
