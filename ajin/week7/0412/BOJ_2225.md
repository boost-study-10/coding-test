### 📖 풀이한 문제

- 백준 2225 - 합분해
- 문제 링크: https://www.acmicpc.net/problem/2225
- 난이도: 골드5
- 걸린 시간: 17분

### 🧩 풀이 설계 / 막힌 부분

- 문제: 0 이상 N 이하의 정수 K개를 더해서 합이 N이 되는 경우의 수 구하기
- 처음 접근 아이디어:
  - i개의 수로 합 j를 만드는 경우를 dp[i][j]로 정의하면,
  - 마지막 수가 0인 경우: 앞의 i - 1개의 수로 합 j를 만들어야하니 dp[i - 1][j]
  - 마지막 수가 1 이상인 경우: i개의 수로 합 j - 1을 만드는 경우이므로 dp[i][j - 1]
- 고려한 시간복잡도:
  - K <= 200이고 N <= 200 이므로 O(K \* N) 가능
- 막힌 부분 / 해결 방법: X

### ⭐️ 문제에서 주로 사용한 알고리즘

- DP

### 🧠 대략적인 코드 설명

1. dp[i][j]를 i개의 수를 더해서 합이 j가 되는 경우의 수로 설정
2. 수 1개로 합 j를 만드는 방법은 항상 j 하나만 고르면 되므로 dp[1][j] = 1로 초기화
3. 합이 0인 경우는 모두 0만 선택하면 되므로 dp[i][0] = 1로 초기화
4. i = 2부터 k까지, j = 1부터 n까지 순회하면서
   dp[i][j] = dp[i - 1][j] + dp[i][j - 1]로 갱신
5. dp[k][n] 출력
