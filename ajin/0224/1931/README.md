### 📖 풀이한 문제

- 백준 1931 - 회의실 배정
- 문제 링크: https://www.acmicpc.net/problem/1931
- 난이도: 골드5
- 걸린 시간: 20분

### ⭐️ 문제에서 주로 사용한 알고리즘

- 그리디

### 🧩 풀이 설계 / 코드 설명

#### 시간 복잡도

1. 부르트포스 → 불가능

- N = 100,000일 때, 각 회의를 선택/미선택하면 O(2^N)

2. 그리디 → 가능
   - 정렬 후 앞에서부터 회의실 들어가기
   - O(N logN) \* O(N)

#### 설계

1. 회의를 끝나는 시간 오름차순으로 정렬
   - 종료 시간이 같으면 시작 시간으로 오름차순 정렬
2. 가장 빨리 끝나는 회의부터 선택
3. 그 다음, 현재 끝난 시간 이후에 시작하는 회의만 선택
   - 회의 개수 1증가 및 최근에 회의 끝난 시간을 현재 회의 종료 시간으로 업데이트

#### 종료 시간으로 정렬하는 이유

| 회의 | 시작 시간 | 종료 시간 |
| ---- | --------- | --------- |
| A    | 1         | 100       |
| B    | 2         | 3         |
| C    | 3         | 4         |

- 시작 시간 기준으로 정렬하면 A를 먼저 고르고 B, C를 선택할 수 없음 -> 오답
- 종료 시간 기준으로 정렬하면 B, C를 고르고 A를 선택 안 함
