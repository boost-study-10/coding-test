### 📖 풀이한 문제

- 백준 1202 - 보석 도둑
- 문제 링크: https://www.acmicpc.net/problem/1202
- 난이도: 골드2
- 걸린 시간: 70분..?!?!?, 60분 이후 정답 확인

### ⭐️ 문제에서 주로 사용한 알고리즘

- 그리디
- 우선순위 큐

### 🧩 풀이 설계 / 코드 설명

#### 시간 복잡도

1. 부르트포스 → 불가능
   - N = 300,000이고 K = 300,000일 때, N\*K 불가능
2. 그리디 → 가능
   - 가방을 순회하면서 보석을 한 번만 스캔하기: O(N + K)
   - 우선순위 큐 사용해야함
   - 보석, 가방 정렬: O(N logN), O(K logK)

#### 설계

1. 보석을 무게 기준 오름차순으로, 가방을 용량 기준 오름차순으로 정렬
2. `jewel_index`로 보석 배열을 딱 한 번만 순회
3. 각 가방 `bag_m`에 대해
   - 해당 가방에 들어갈 수 있는 보석들의 가격을 PQ에 넣기
   - 특정 가방에 보석 다 넣었다면
     - 현재 가방에 넣을 보석은 후보 중 가장 비싼 것 1개 를 PQ에서 꺼내서 정답에 더하기
