### 📖 풀이한 문제

- 백준 1987 - 알파벳
- 문제 링크: https://www.acmicpc.net/problem/1987
- 난이도: 골드4
- 걸린 시간: 30분 / 2트

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어: DFS로 이동하면서 방문한 알파벳을 Set에 기록하고, 이미 사용된 알파벳이면 이동하지 않기(백트래킹)
- 고려한 시간복잡도:
  - R = 20, C = 20, 알파벳 최대 26개 일 때, DFS 깊이가 26까지라면 O(4^26)
  - 그래도 같은 알파벳이 나오면 탐색 종료하므로 pruning 처리되어 가능
- 막힌 부분 / 해결 방법:
  - 백트래킹해야하는 데, 까먹고 안해버림..

### ⭐️ 문제에서 주로 사용한 알고리즘

- DFS(백트래킹)
  - 가능한 경로를 탐색하면서 재귀로 이전 방문 기록 복구 가능
  - BFS는 큐에 방문한 알파벳 집합을 포함하고 있어야해서 비효율적이라 판단

-> 가능한 경로 찾기 문제는 DFS(백트래킹)가 유리하다!

### 🧠 대략적인 코드 설명

- 알파벳을 한 번만 이용 가능하므로 Set() 으로 방문 관리
  - 일반 배열을 사용하면 index와 알파벳의 관계를 정의해야하므로 귀찮아진다..
  - 알파벳 자체를 사용했는지 안했는지 구분하기위해 Set 사용

1. 좌측 상단 시작점부터 백트래킹 돌기
   1. 이동 가능한 경로인지 확인
   2. visited에 해당 알파벳 추가하고 개수 증가
   3. DFS 방문
   4. DFS 종료 후 visited 되돌리기
2. 최대 길이 경로인지 매번 개수 업데이트
