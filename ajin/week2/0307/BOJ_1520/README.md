### 📖 풀이한 문제

- 백준 1520 - 내리막 길
- 문제 링크: https://www.acmicpc.net/problem/1520
- 난이도: 골드3
- 걸린 시간: 30분+ / 2트 / 정답 확인

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어: 시작점에서 도착점까지 내리막길로만 이동하는 모든 경로를 DFS로 탐색
- 고려한 시간복잡도:
  - N = 500, M = 500일 때, 250,000칸
  - 단순 DFS로 모든 경로 탐색하면 같은 칸에서 출발하는 하위 경로를 반복 계산 ..?
  - DP 사용해야 O(N \* M) 가능
- 막힌 부분 / 해결 방법:
  - 특정 위치에서 도착점까지 가는 경로를 visited에 저장(DP)해 pruning

### ⭐️ 문제에서 주로 사용한 알고리즘

- DFS + DP
  - 각 칸마다 최대 4방향 확인
    - 이미 방문한 칸이라면 O(1)
    - 처음 방문한 칸이라면 상하좌우 4칸 확인
  - visited[x][y] = (x, y)에서 도착점까지 갈 수 있는 경로 수
    - 현재 칸의 경로 수 = 이동 가능한 다음 칸들의 경로 수 합

### 🧠 대략적인 코드 설명

1. 시작점(0,0)에서 DFS 탐색
   - 현재 칸이 도착점(M-1, N-1)이면 경로 1개이므로 `1` 반환
   - 이미 계산한 칸이면 저장된 값 반환
   - 아직 계산하지 않은 칸이면:
     1. visited `0`으로 초기화
     2. 상하좌우 중 더 낮은 칸으로 이동
     3. 이동 가능한 칸들의 `dfs(nx, ny)` 값을 모두 더하기
