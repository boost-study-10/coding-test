### 📖 풀이한 문제

- 백준 1072 - 게임
- 문제 링크: https://www.acmicpc.net/problem/1072
- 난이도: 실버3
- 걸린 시간: 33분 / 2트 / 반례 참고

### 🧩 풀이 설계 / 막힌 부분

- 문제:
  - 전체 게임 수 X와 이긴 게임 수 Y가 주어질 때,
  - 승률이 처음으로 변하기 위해 필요한 최소 추가 게임 수 구하기
- 처음 접근 아이디어:
  - 승률은 게임을 한 판씩 늘릴 때마다 승률 변화만 비교하면 되니까
  - 이분 탐색으로 승률이 변화되는 경계값 찾기
  - 어떤 cnt에 대해 승률이 증가했다면, 정답 후보로 저장하고 더 작은 쪽을 탐색해야 함
  - 반대로 아직 승률이 그대로라면 더 많은 게임이 필요하므로 오른쪽 구간을 탐색
- 고려한 시간복잡도:
  - 게임 을 최대 1,000,000,000까지 할 수 있으니 O(N) 체크 불가능
  - 이분 탐색을 사용하면 O(logN)에 승률 비교 가능
  - 각 탐색마다 승률 계산은 O(1)이므로 전체 시간복잡도는 O(log 1,000,000,000)
- 막힌 부분 / 해결 방법:
  - Math.floor(Y / X \* 100) 로 승률 계산 시 JS에서 숫자를 부동소수점으로 계산하므로 Y / X 과정에서 소수 오차가 생길 수 있음
  - 예를 들어 원래 정답이 80이어야 하는데, 내부 계산이 79.999999999...처럼 나오면 Math.floor 결과가 79가 되어 버릴 수 있음
  - 그래서 Math.floor(Y \* 100 / X)처럼 100을 먼저 곱해서 최대한 정수 계산에 가깝게 만든 뒤 나누는 방식이 안전함

### ⭐️ 문제에서 주로 사용한 알고리즘

- 이분탐색

### 🧠 대략적인 코드 설명

1. 전체 게임 수 X, 이긴 게임 수 Y에 대해 현재 승률 Z = Math.floor(Y \* 100 / X) 계산
2. 이분 탐색 중간값 cnt를 잡고, cnt판을 더 이겼다고 가정한 새로운 승률 z 계산
   - 만약 새로운 승률 z가 기존 승률 Z보다 크다면
     정답 후보가 될 수 있으므로 answer = cnt
     더 작은 값이 있는지 확인하기 위해 왼쪽 구간 탐색
   - 만약 새로운 승률 z가 Z보다 작거나 같다면
     아직 승률이 변하지 않았으므로 더 많은 게임이 필요하니 오른쪽 구간 탐색
3. 탐색이 끝난 뒤에도 정답이 갱신되지 않았다면 -1, 아니면 answer 출력
