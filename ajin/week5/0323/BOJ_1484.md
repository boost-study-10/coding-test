### 📖 풀이한 문제

- 백준 1484 - 다이어트
- 문제 링크: https://www.acmicpc.net/problem/1484
- 난이도: 골드5
- 걸린 시간: 14분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 문제: 현재 몸무게의 제곱에서 이전 몸무게의 제곱을 뺐을 때 G가 되는 현재 몸무게 구하기
- 처음 접근 아이디어:
  - G = 현재몸무게^2 - 이전몸무게^2 이니까 가능한 몸무게 후보를 오름차순으로 두고 투포인터로 탐색
- 고려한 시간복잡도:
  - G가 100,000이고, `50001^2 - 50000^2 = 100001` 이므로,
  - 현재 몸무게 후보는 최대 50,000까지만 확인해도 됨
  - 투포인터는 left와 right가 각각 최대 50,000까지 한 번씩만 증가하므로 전체 시간복잡도는 O(MAX_N), 즉 O(50,000)
- 막힌 부분 / 해결 방법:
  - 최대 제곱수를 직접 구하기는 어려우니 빠르게 브루트포스돌려서 구함
  ```js
  for (let i = 1; i < 1000000; i++) {
    if (100000 < (i + 1) * (i + 1) - i * i) {
      console.log(i, (i + 1) * (i + 1), i * i);
      break;
    }
  }
  ```

### ⭐️ 문제에서 주로 사용한 알고리즘

- 투포인터

### 🧠 대략적인 코드 설명

1. 몸무게 후보의 최댓값을 50,000으로 설정
2. `left = 1`, `right = 2`로 투포인터 탐색
   - `right^2 - left^2` 값을 계산
   - 계산한 값이
     - G보다 작으면 현재 몸무게를 늘려야 하므로 right++
     - G보다 크면 이전 몸무게를 늘려 차이를 줄이기 위해 left++
     - G와 같으면 right를 정답 배열에 추가하고 다음 경우를 보기 위해 left++
3. 탐색이 끝난 뒤
   - 정답이 없으면 -1
   - 있으면 오름차순으로 출력
