### 📖 풀이한 문제

- 백준 1477 - 휴게소 세우기
- 문제 링크: https://www.acmicpc.net/problem/1477
- 난이도: 골드4
- 걸린 시간: 50분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 문제:
  - 고속도로 길이 L 위에 기존 휴게소들이 주어질 때,
  - 휴게소 M개를 더 세운 뒤 휴게소가 없는 구간의 최대 길이의 최솟값 구하기
- 처음 접근 아이디어:
  - 처음에는 가장 긴 구간을 절반씩 나누는 식의 그리디를 떠올림
  - 하지만 어떤 구간은 두 개가 아니라 세 개 이상으로 나누는 것이 최적일 수 있어서,
  - 단순히 현재 가장 긴 구간만 잘라서는 전체 최적해를 보장할 수 없음
  - 휴게소 위치 자체를 직접 정하는 대신,
  - 가능한 최대 구간 길이를 기준으로 이분 탐색 진행
- 고려한 시간복잡도:
  - N <= 50, M <= 100, L <= 1000일 때,
  - 최대 구간 길이 후보는 1 ~ L - 1 범위에서 이분 탐색
  - 각 후보 길이에 대해, 기존 구간들을 해당 길이 이하로 만들기 위해 필요한 휴게소 수를 계산하는 데 O(N)이므로
  - 전체 시간복잡도는 O(N log L) = O(50 \* log 1000)
- 막힌 부분 / 해결 방법: 처음에 그리디로 접근해서 시간이 오래걸림

### ⭐️ 문제에서 주로 사용한 알고리즘

- 이분탐색

### 🧠 대략적인 코드 설명

1. 기존 휴게소 위치와 시작점 0과 끝점 L을 오름차순 정렬
2. 인접한 휴게소 사이의 거리들을 gap에 저장
3. 이분 탐색으로 가능한 최대 구간 길이 max_gap의 최솟값 탐색
4. 중간값 gap에 대해 현재 모든 구간을 이 길이 이하로 만들기 위해 필요한 휴게소 수 계산
   - 필요한 휴게소 수가 M 이하라면 현재 max_gap으로도 가능하므로 더 작은 값 탐색하고 정답 업데이트
   - 필요한 휴게소 수가 M보다 크다면 현재 max_gap이 너무 작으므로 더 큰 값 탐색
