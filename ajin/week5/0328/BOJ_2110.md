### 📖 풀이한 문제

- 백준 2110 - 공유기 설치
- 문제 링크: https://www.acmicpc.net/problem/2110
- 난이도: 골드4
- 걸린 시간: 40분 / 2트 / 1트 후 친구와 논의 ㅎ.ㅎ

### 🧩 풀이 설계 / 막힌 부분

- 문제:
  - N개의 집에 공유기 C개를 설치할 때, 가장 인접한 두 공유기 사이 거리의 최댓값 구하기
- 처음 접근 아이디어:
  - 집을 정렬한 뒤, 공유기 사이의 최소 거리를 하나의 값으로 정해 이분탐색
  - 어떤 최소 거리 K가 주어졌을 때, 공유기를 greedy하게 설치해서 C개 **이상** 놓을 수 있는지 확인
  - 가능하다면 더 큰 거리도 시도하고, 불가능하다면 거리를 줄이는 방식으로 탐색
- 고려한 시간복잡도:
  - N <= 200,000, 위치 <= 2^31 이므로 공유기 설치 조합을 직접 시도하는 방식은 불가능
  - 집 정렬 O(N log N), 이분탐색 O(log 2^31) = 31, 각 탐색마다 공유기 설치 가능 여부 확인 O(N)
  - 전체 시간복잡도는 O(N log N + N log D)
- 막힌 부분 / 해결 방법: 틀린 이유
  - 처음에는 최소 거리 K로 공유기를 설치했을 때, 설치 가능한 개수 M이 정확히 C개일 때만 정답 후보로 갱신해서 틀림
  - 어떤 거리 K에서 공유기를 C + 1개 설치할 수 있다면, 그중 일부만 선택해서 C개만 설치하는 것도 가능하므로
  - M >= C라면 해당 거리 K는 정답 후보

### ⭐️ 문제에서 주로 사용한 알고리즘

- 이분탐색, 그리디

### 🧠 대략적인 코드 설명

1. 집 위치를 오름차순 정렬
2. 공유기 사이 최소 거리의 탐색 범위를 1 ~ 가장 큰 위치로 이분탐색 진행
   - mid는 현재 시도할 최소 거리
3. 공유기 사이 거리를 mid 이상 유지하며 몇 개까지 설치 가능한지 계산
   - 설치 가능한 개수가 C 이상이면 현재 거리도 가능하므로 정답 후보 갱신
     더 큰 거리도 가능한지 보기 위해 left = mid + 1
   - 설치 가능한 개수가 C보다 작으면 현재 거리는 너무 크므로 right = mid - 1
4. 가능한 최소 거리의 최댓값 출력
