### 📖 풀이한 문제

- 백준 1920 - 수 찾기
- 문제 링크: https://www.acmicpc.net/problem/1920
- 난이도: 실버4
- 걸린 시간: 10분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 문제:
  - N개의 자연수로 이루어진 수열 A가 주어질 때,
  - M개의 수가 각각 A에 존재하는지 구하기
- 처음 접근 아이디어:
  - 수열 A를 정렬한 뒤, 찾고 싶은 수마다 이분탐색
- 고려한 시간복잡도:
  - N, M <= 100,000이므로 완전탐색으로 찾는 O(N \* M)은 불가능
  - 수열 A를 정렬하는 데 O(N log N), M개의 수에 대해 각각 이분탐색을 하면 O(M log N)
  - 전체 시간복잡도 O(N log N + M log N)
- 막힌 부분 / 해결 방법: X

### ⭐️ 문제에서 주로 사용한 알고리즘

- 이분탐색
- 수열 A를 Set 또는 Map으로 저장한 뒤 has로 검사하는 방식도 가능

### 🧠 대략적인 코드 설명

1. arr를 오름차순 정렬
2. nums의 각 값에 대해 이분탐색 진행
   - 중간값 mid를 기준으로 목표값과 비교
   - 목표값이 더 크면 오른쪽 구간 탐색
   - 목표값이 더 작으면 왼쪽 구간 탐색
3. 같으면 1 반환하고 끝까지 찾지 못하면 0 반환
