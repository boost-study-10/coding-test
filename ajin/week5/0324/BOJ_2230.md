### 📖 풀이한 문제

- 백준 2230 - 수 고르기
- 문제 링크: https://www.acmicpc.net/problem/2230
- 난이도: 골드5
- 걸린 시간: 27분 / 4트 / 디버깅 20분.....

### 🧩 풀이 설계 / 막힌 부분

- 문제: 정수 수열에서 두 수를 골랐을 때, 차이가 M 이상이 되는 경우 중 가장 작은 차이 구하기
- 처음 접근 아이디어:
  - 수열을 정렬한 뒤 투포인터 탐색
  - 차이가 M 이상이면 정답을 갱신하고 더 작은 차이를 찾기 위해 left를 증가
  - 차이가 M보다 작으면 차이를 키워야 하므로 right를 증가
- 고려한 시간복잡도:
  - N이 100,000이므로 모든 시작점마다 끝점을 다시 찾는 O(N^2)은 불가능
  - 정렬 O(NlogN), 투포인터를 사용하면 left, right가 각각 한 번씩만 증가하므로 O(N)
  - 전체 시간복잡도는 O(NlogN)
- 막힌 부분 / 해결 방법:
  - 첫번째 오답: 최대 차 범위 잘못 설정함
  - 두번째 오답: left, right 조건 설정 안함
  - 세번째 오답: 입력 마지막에 공백이 있어 0값이 들어감

### ⭐️ 문제에서 주로 사용한 알고리즘

- 투포인터

### 🧠 대략적인 코드 설명

1. 입력값을 받고 수열을 오름차순으로 정렬
2. left = 0, right = 0으로 투포인터 탐색 시작
   - 현재 두 수의 차이 diff를 계산
   - M < diff라면 정답 후보가 될 수 있으므로 answer를 갱신, 더 작은 차이도 가능한지 보기 위해 left++
   - diff< M라면 차이를 더 키워야 하므로 right++
3. answer 출력
