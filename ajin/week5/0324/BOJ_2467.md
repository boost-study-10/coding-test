### 📖 풀이한 문제

- 백준 2467 - 용액
- 문제 링크: https://www.acmicpc.net/problem/2467
- 난이도: 골드5
- 걸린 시간: 12분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 문제: 두 용액의 합이 0과 가까운 두 용액 구하기
- 처음 접근 아이디어:
  - 수열을 정렬한 뒤 양 끝에서 투포인터 탐색
  - 두 용액의 합이 0보다 작은지 큰지에 따라 포인터를 이동
  - 합의 절댓값이 작을수록 0에 가까우므로, 각 후보의 절댓값을 저장해두고
  - 정렬한 후 가장 작은 경우를 선택
- 고려한 시간복잡도:
  - N이 100,000이므로 모든 시작점마다 끝점을 다시 찾는 O(N^2)은 불가능
  - 정렬 O(NlogN), 투포인터를 사용하면 left, right가 각각 한 번씩만 증가하므로 O(N)
  - 전체 시간복잡도는 O(NlogN)
- 막힌 부분 / 해결 방법: X

### ⭐️ 문제에서 주로 사용한 알고리즘

- 투포인터

### 🧠 대략적인 코드 설명

1. 용액을 오름차순 정렬
2. left = 0, right = N - 1로 투포인터 탐색 시작
   - 현재 두 용액의 합 arr[left] + arr[right]를 계산
   - 합의 절댓값과 현재 포인터 위치를 후보 배열에 저장
   - 합이 0보다 작으면 더 큰 값이 필요하므로 left++
   - 0보다 크면 더 작은 값이 필요하므로 right--
   - 0이면 가장 좋은 경우이므로 종료
3. 저장한 후보들을 합의 절댓값 기준으로 정렬
4. 가장 앞에 있는 후보의 두 인덱스로 접근해 정답 출력
