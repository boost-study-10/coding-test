### 📖 풀이한 문제

- 백준 1644 - 소수의 연속합
- 문제 링크: https://www.acmicpc.net/problem/1644
- 난이도: 골드3
- 걸린 시간: 25분 / 4트 / 테스트케이스 검증!!!!!

### 🧩 풀이 설계 / 막힌 부분

- 문제: N을 연속된 소수의 합으로 나타낼 수 있는 경우의 수 구하기
- 처음 접근 아이디어:
  - 먼저 N 이하의 모든 소수를 구하고
  - 소수 배열에서 연속된 구간의 합이 N이 되는 경우를 투포인터로 탐색
  - 연속된 수열의 합을 다루는 문제이므로, 매번 구간합을 다시 계산하지 않고 현재 합을 유지하며 포인터를 이동
- 고려한 시간복잡도:
  - N이 4,000,000이므로 소수를 단순 판별로 하나씩 찾으면 비효율적
  - 에라토스테네스의 체로 소수를 구하면 대략 O(Nlog(log N))
  - 소수 배열에 대해 투포인터 탐색은 각 포인터가 한 번씩만 이동하므로 O(N)
- 막힌 부분 / 해결 방법:
  - 첫번째 오답: 2 + 2처럼 같은 소수를 여러 번 더하는 경우를 잘못 포함했음 / 문제 잘 읽기 ㅎㅎ
  - 두번째 오답: 소수를 찾는 과정에서 경계값 처리를 잘못해서 일부 수가 잘못 판별되었음 / for문에서 경계값 체크 잘하기
  - 세번째 오답: 입력값에 공백 처리 안함 / trim 까먹지 말자...

### ⭐️ 문제에서 주로 사용한 알고리즘

- 투포인터
- 에라토스테네스의 체(잊을만하면 한번 씩 나옴)

### 🧠 대략적인 코드 설명

1. 에라토스테네스의 체로 N 이하의 소수 배열 prime 생성
2. N 자체가 소수라면 길이 1짜리 연속합이 가능 N이 소수인지 확인
3. left = 0, right = 1로 두고 투포인터 탐색
   - 현재 합 sum이 N과 같으면 정답 개수 증가
   - N 이하이면 더 긴 구간을 보기 위해 right++
   - N보다 크면 구간합을 줄이기 위해 left++

### 에라토스테네스의 체

```js
function get_prime() {
  const prime = Array(N + 1).fill(true);
  prime[0] = prime[1] = false;

  for (let i = 2; i * i < N + 1; i++) {
    if (!prime[i]) continue;
    for (let j = i * i; j < N + 1; j += i) {
      prime[j] = false;
    }
  }

  return prime.flatMap((isPrime, i) => (isPrime ? [i] : []));
}
```
