### 📖 풀이한 문제

- 백준 2473 - 세 용액
- 문제 링크: https://www.acmicpc.net/problem/2473
- 난이도: 골드3
- 걸린 시간: 30분 / 3트 / 메모리초과..?

### 🧩 풀이 설계 / 막힌 부분

- 문제: 합이 0에 가장 가까운 세 용액 구하기
- 처음 접근 아이디어:
  - 용액 배열을 오름차순 정렬
  - 한 용액을 기준값으로 고정한 뒤, 나머지 구간에서 투포인터로 두 용액 선택
- 고려한 시간복잡도:
  - N이 5,000이므로 세 수를 모두 직접 고르는 O(N^3)은 불가능
  - 하나를 고정하고 나머지 두 수를 투포인터로 찾으면 O(N^2)에 해결 가능
  - 정렬 O(N log N), 이후 각 기준값마다 투포인터 O(N), 전체 시간복잡도는 O(N^2)
- 막힌 부분 / 해결 방법:
  - 가능한 후보 조합들을 Map 등에 모두 저장해두고, 마지막에 정렬해서 답을 구하면
  - N이 5,000이라 후보가 많아질 수 있어서, 메모리가 크게 증가함
  - 탐색 중마다 현재 합의 절댓값을 확인하고, 기존 최솟값보다 더 좋으면 바로 정답을 갱신하는 방식으로 변경

### ⭐️ 문제에서 주로 사용한 알고리즘

- 투포인터

### 🧠 대략적인 코드 설명

1. 입력받은 용액 배열을 오름차순 정렬
2. 배열의 각 원소를 하나씩 기준값 fixed로 선택
3. 기준값보다 오른쪽 구간에서 left = idx + 1, right = N - 1로 투포인터 탐색
   - 세 용액의 합 fixed + arr[left] + arr[right] 계산
     - 현재 합의 절댓값이 best보다 작으면 best 갱신하고 현재 세 용액 값을 answer에 저장
   - 합이 0보다 작으면 더 큰 값이 필요하므로 left++
   - 0보다 크면 더 작은 값이 필요하므로 right--
   - 0이면 가장 이상적인 경우이므로 종료
4. 모든 기준값에 대해 반복한 뒤 최종 정답 출력
