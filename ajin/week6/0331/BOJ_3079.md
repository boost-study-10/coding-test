### 📖 풀이한 문제

- 백준 3079 - 입국심사
- 문제 링크: https://www.acmicpc.net/problem/3079
- 난이도: 골드5
- 걸린 시간: 29분 / 3트 / 정답 참고

### 🧩 풀이 설계 / 막힌 부분

- 문제: N개의 심사대가 있을 때, M명이 모두 입국심사를 받는 데 걸리는 최소 시간 구하기
- 처음 접근 아이디어:
  - 시간을 기준으로 이분탐색
  - 특정 시간 T가 주어졌을 때, 각 심사대가 T 시간 동안 몇 명을 처리할 수 있는지 계산
  - 처리 가능한 총 인원이 M명 이상이면 더 짧은 시간도 가능한지 확인하고, 부족하면 시간을 늘리기
- 고려한 시간복잡도:
  - 심사받는 사람 수 M <= 1,000,000,000이므로 한 명씩 시뮬레이션하는 방식은 불가능
  - 최소 시간을 이분탐색하면 가능한 시간 범위는 1 ~ 가장 느린 심사대 시간 \* M 이고 O(log(maxTime)), 각 탐색마다 모든 심사대를 확인하므로 O(N)
  - 전체 시간복잡도는 O(N log(maxTime))
- 막힌 부분 / 해결 방법:
  - 틀린 이유:
    - 이 문제에서 가능한 최댓값은 M \* time[N - 1]
    - 문제 조건상 M <= 10^9, 심사 시간도 최대 10^9이므로 최악의 경우 10^18까지 갈 수 있음
    - JS의 Number는 정수를 정확하게 표현할 수 있는 범위가 2^53 - 1까지라서, 10^18 수준의 값을 정확히 다루지 못함
    - 그래서 시간 범위, 중간값, 처리 인원 수를 모두 BigInt로 바꿔서 해결
  - BigInt 나눗셈은 소수점을 만들지 않고 자동으로 정수 몫만 반환함
  - 항상 탐색 범위가 안전한 정수 범위를 넘는지 확인하기!!!

### ⭐️ 문제에서 주로 사용한 알고리즘

- 이분탐색

### 🧠 대략적인 코드 설명

1. 최악의 경우 걸리는 최대 시간 MAX_TIME = M \* 가장 느린 심사 시간 계산
2. 최소 시간을 구하기위해 1 ~ MAX_TIME 범위로 이분탐색 진행

- 시간 안에 총 몇 명을 처리할 수 있는지 계산
  - 각 심사대마다 mid / t명을 처리 가능
  - 모두 더해 총 인원 수 계산
- 처리 가능한 인원이 M 이상이면 현재 시간 안에 모두 심사 가능하므로 정답 후보 갱신하고 더 짧은 시간도 가능한지 보기 위해 right = mid - 1
- 처리 가능한 인원이 M보다 작으면 현재 시간이 부족하므로 left = mid + 1
