### 📖 풀이한 문제

- 백준 2143 - 두 배열의 합
- 문제 링크: https://www.acmicpc.net/problem/2143
- 난이도: 골드 3
- 걸린 시간: 45분

### 🧩 풀이 설계 / 막힌 부분

- 문제: 배열 A, B의 연속 부분배열 합을 각각 하나씩 골라 더했을 때, 그 합이 T가 되는 경우의 수 구하기
- 처음 접근 아이디어:
  - 배열 A와 B에서 만들 수 있는 모든 연속 부분합을 먼저 구하고
  - A의 부분합 하나를 고르면, B에서는 T - a인 부분합이 몇 개 있는지 찾기
  - 부분합은 모두 정렬하고 이분 탐색으로 T - a의 개수를 세기
- 고려한 시간복잡도:
  - 배열 길이 <= 1000일 때,
  - 연속 부분합을 구하는 데 각각 O(n^2), O(m^2)
  - 연속 부분합 개수 n(n+1)/2, m(m+1)/2를 정렬하는 데 O(n^2 log n), O(m^2 log m)
  - A의 각 부분합마다 B에서 이분 탐색을 두 번 수행하면 O(n^2 + m^2 + n^2 log n^2 + m^2 log m^2 + n^2 log m^2)
- 막힌 부분 / 해결 방법: X

### ⭐️ 문제에서 주로 사용한 알고리즘

- 누적 합

### 🧠 대략적인 코드 설명

1. 배열 A, B의 모든 연속 부분합을 Asum, Bsum에 저장하고 각각 오름차순 정렬
2. Asum의 각 값 a에 대해, B에서 필요한 값 b = T - a 계산 후
   1. bisect_B_left(b)로 b가 처음 등장하는 위치를 찾고
   2. bisect_B_right(b)로 b가 마지막으로 등장하는 위치를 찾아
   3. right - left + 1만큼 정답에 더해, a와 짝이 되는 B의 부분합 개수 더하기

### 다른 풀이

> A의 모든 연속 부분합을 Map에 저장해, 부분합 값별 등장 횟수를 관리

1. A의 연속 부분합을 만들면서 Map으로 개수 저장
2. B의 모든 연속 부분합을 구하면서
   1. 매번 T - s가 A의 부분합에 몇 번 등장했는지만 확인

- 시간복잡도는 A의 부분합 생성 O(n^2), B의 부분합 생성 O(m^2)하면 O(n^2 + m^2)
