### 📖 풀이한 문제

- 백준 1507 - 궁금한 민호
- 문제 링크: https://www.acmicpc.net/problem/1507
- 난이도: 골드2
- 걸린 시간: 40분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 문제: 모든 도시 쌍 사이의 최단 이동 시간이 주어졌을 때, 필요한 도로만 남겼을 때의 총 시간 합 구하기
- 처음 접근 아이디어:
  - 주어진 표는 이미 모든 도시 쌍 사이의 최단 거리이므로
  - 어떤 두 도시 i, j 사이의 도로가 실제로 필요한지 보려면
  - 다른 도시 k를 경유해서도 같은 최단 거리를 만들 수 있는지 확인하면 됨
  - 만약 i -> k -> j가 i -> j와 같다면, i -> j 도로는 없어도 최단 거리표를 유지할 수 있으므로 제거하면 됨
  - 추가로 i -> k -> j가 i -> j보다 더 짧다면, 주어진 거리표 자체가 모순이므로 잘못된 입력 처리하기
- 고려한 시간복잡도:
  - N <= 20이라서 모든 도시 쌍과 경유지를 전부 확인해도 가능
  - i, j, k를 기준으로 검사하므로 전체 시간복잡도는 O(N^3)
  - 문제에서 도로 시간이 대칭이여서 i < j인 경우만 보면 되므로 2차원 배열의 절반만 확인하면 됨
- 막힌 부분 / 해결 방법:
  - 직전에 플로이드 와샬 문제를 풀었는데도 아이디어 떠올리기 쉽지 않았다..

### ⭐️ 문제에서 주로 사용한 알고리즘

- 플로이드 와샬
  - 모든 정점 쌍이 최단 거리라면 dist[i][j] <= dist[i][k] + dist[k][j]

### 🧠 대략적인 코드 설명

1. i < j인 도시 쌍만 확인하면서 도로가 실제로 필요한지 확인하기
2. 각 도시 쌍 (i, j)에 대해 모든 다른 도시 k를 경유지 두기
   - roads[i][k] + roads[k][j] < roads[i][j]라면 주어진 거리표보다 더 짧은 경로가 존재한다는 뜻이므로 즉시 -1 반환
   - roads[i][k] + roads[k][j] === roads[i][j]인 경우가 하나라도 있으면 i -> j 도로는 다른 경로로 대체 가능(필요한 도로가 아님)
   - 어떤 경유지로도 같은 거리를 만들 수 없다면 i -> j 도로는 반드시 필요하므로 합에 추가
3. 모든 도시 쌍을 검사한 뒤 필요한 도로들의 총합 반환
