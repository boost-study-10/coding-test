### 📖 풀이한 문제

- 백준 1865 - 웜홀
- 문제 링크: https://www.acmicpc.net/problem/1865
- 난이도: 골드3
- 걸린 시간: 45분 / 2트 / 정답 참고

### 🧩 풀이 설계 / 막힌 부분

- 문제: 한 지점에서 출발해 도로와 웜홀을 지난 뒤 다시 출발 지점으로 돌아왔을 때, 시간이 처음보다 줄어드는 경우가 있는지 구하기
- 처음 접근 아이디어:
  - 일반 도로는 양방향이고 가중치가 양수, 웜홀은 단방향이고 가중치가 음수이므로
  - 음수 사이클 판별이 가능한 벨만-포드 알고리즘 사용
  - 문제에서 출발 지점이 정해져 있지 않으므로, 모든 정점에서 시작할 수 있도록 0번 정점을 추가하고 모든 정점으로 가중치 0 간선을 연결
- 고려한 시간복잡도:
  - 테스트케이스 수 TC <= 5, 정점 수 N <= 500, 간선 수 M <= 2500, 웜홀 수 W <= 200일 때,
  - 벨만-포드는 정점 수만큼 모든 간선을 확인하므로 O(N \* E)
  - E가 최대 2 \* 2500 + 200 + 500 = 5700이므로 충분히 가능
- 막힌 부분 / 해결 방법:
  - 벨만 포드에서 모든 정점에서 출발해야 한다 조건을 구현으로 옮기는 부분이 어려웠음
  - 답지 참고 후 0번 가상 정점을 만들어 해결

### ⭐️ 문제에서 주로 사용한 알고리즘

- 벨만포드

### 🧠 대략적인 코드 설명

1. 일반 도로는 양방향으로 저장하고 웜홀은 단방향으로 가중치를 음수로 바꿔 저장
2. 모든 정점에서 시작할 수 있도록 0번 정점에서 각 정점으로 가는 비용 0 간선 추가
3. 벨만 포드 돌면서 음수 사이클 존재하는지 확인하기
   - 거리 배열을 Infinity로 초기화하고 시작점 0의 거리를 0으로 설정
   - 총 N(N + 1 - 1)개 정점에 대해 최단 거리 업데이트
   - 만약 N + 1 반복에서 값이 업데이트된다면, 음수 사이클 존재이므로 "YES" 반환
4. 끝에서 추가 갱신이 없으면 "NO" 반환
