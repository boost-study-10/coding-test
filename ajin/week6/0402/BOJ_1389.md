### 📖 풀이한 문제

- 백준 1389 - 케빈 베이컨의 6단계 법칙
- 문제 링크: https://www.acmicpc.net/problem/1389
- 난이도: 실버 1
- 걸린 시간: 21분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 문제: 각 유저에 대해 다른 모든 유저까지의 최단 친구 거리 합이 가장 작은 유저 찾기
- 처음 접근 아이디어:
  - 각 유저를 시작점으로 BFS를 돌려 다른 유저까지의 거리를 구하고
  - 시작 유저에서 각 유저까지의 거리 합을 계산한 뒤, 합이 가장 작은 사람 선택
- 고려한 시간복잡도:
  - N <= 100, M <= 5,000이므로 모든 유저에 대해 BFS를 한 번씩 수행하면, O(N(N + M))
- 막힌 부분 / 해결 방법: X
- 다른 풀이:
  - 모든 사람 쌍 사이의 최단 거리를 알고 싶어지는 문제이고 N이 작기 때문에 플로이드 와샬 사용하는 게 정석
  - 플로이드 와샬은 모든 정점 쌍 최단 거리를 구하므로 O(N^3) = 1,000,000 가능
  -

### ⭐️ 문제에서 주로 사용한 알고리즘

- BFS
  - 모든 간선의 가중치가 1이므로 가능
- 플로이드 와샬
  - N이 작을 때, 모든 정점 쌍 사이의 최단 거리를 구하는 알고리즘

### 🧠 대략적인 코드 설명

1. 친구 관계를 인접 리스트 friends에 양방향으로 저장하고
2. 시작 유저에서 각 유저까지의 거리를 저장할 2차원 배열(kevin_bacon[start][target]) 생성
3. 1번 유저부터 N번 유저까지 각각 BFS 수행
   - 현재 유저와 연결된 친구들을 순회하면서 아직 방문하지 않은 유저의 거리를 갱신
4. BFS가 끝나면 시작 유저에서 다른 모든 유저까지의 거리 합 계산
5. 거리 합이 가장 작은 유저를 정답으로 갱신하고 케빈 베이컨 수가 가장 작은 유저를 정답으로 선택

### 🧠 플로이드와샬 코드 설명

1. i번 유저에서 j번 유저까지의 최단 거리인 2차원 배열(kevin_bacon[i][j]) 생성
2. 처음에는 자기 자신으로 가는 거리는 0, 직접 친구인 경우는 1, 연결이 아직 없으면 Infinity로 초기화
3. k를 경유지로 두고, 모든 i -> j 경로에 대해
   - 기존 거리(i -> j)와 i -> k -> j로 가는 거리 중 더 작은 값으로 갱신
   - 이 과정을 모든 k에 대해 반복하면 모든 유저 쌍 사이의 최단 거리가 완성됨
4. 이후 각 유저 행의 거리 합을 구해서, 합이 가장 작은 유저를 정답으로 선택
