### 📖 풀이한 문제

- 백준 2211 - 네트워크 복구
- 문제 링크: https://www.acmicpc.net/problem/2211
- 난이도: 골드2
- 걸린 시간: 28분

### 🧩 풀이 설계 / 막힌 부분

- 문제: 1번 컴퓨터를 기준으로 모든 컴퓨터가 통신 가능하도록 복구할 때, 필요한 최소 회선 정보 구하기
- 처음 접근 아이디어:
  - 1번 컴퓨터에서 각 컴퓨터까지의 최단 거리를 구하면서
  - 현재 컴퓨터에 오기 직전의 컴퓨터를 함께 저장하기
  - 회선 비용이 모두 양수이므로 다익스트라 사용
- 고려한 시간복잡도:
  - N <= 1,000이고 M은 주어지지 않았지만, 우선순위 큐를 사용한 다익스트라는 O((N + M) log N)
- 막힌 부분 / 해결 방법: X

### ⭐️ 문제에서 주로 사용한 알고리즘

- 최단거리(다익스트라)

### 🧠 대략적인 코드 설명

1. 각 회선을 양방향 인접 리스트 lines에 저장
2. 다익스트라를 수행하면서 visited에 1번 컴퓨터에서 i번 컴퓨터까지의 최소 시간 저장
   - 우선순위 큐에는 [현재까지 시간, 현재 컴퓨터, 이전 컴퓨터] 형태로 삽입
   1. 큐에서 가장 비용이 적은 컴퓨터를 꺼냄
   2. 이미 더 짧은 경로가 기록되어 있으면 무시
   3. 그렇지 않으면 현재 컴퓨터와 이전 컴퓨터의 연결 정보를 restore_lines에 저장
   4. 현재 컴퓨터와 연결된 다음 컴퓨터들을 순회하면서 더 짧은 경로를 찾으면 거리 갱신하고 현재 컴퓨터를 이전 컴퓨터로 하여 큐에 삽입
3. 탐색이 끝나면 시작점 1번을 제외한 복구 회선 목록 출력

## 다른 풀이

1. 경로를 구할 때, parent 리스트를 별도로 두기
2. 다익스트라를 돌면서 더 짧은 경로를 찾으면 경로와 함께 parent 갱신
   - parent[nextComputer] = computer
3. 2번 컴퓨터부터 현재 컴퓨터와 부모 컴퓨터 출력
