### 📖 풀이한 문제

- 백준 2805 - 나무 자르기
- 문제 링크: https://www.acmicpc.net/problem/2805
- 난이도: 실버2
- 걸린 시간: 11분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 문제: 절단기 높이 H를 정했을 때, 잘린 나무 길이의 합이 M 이상이 되도록 하는 최대 높이 구하기
- 처음 접근 아이디어:
  - 절단기 높이를 하나 정하고, 그 높이로 잘랐을 때 얻을 수 있는 나무 길이 합 계산
  - 잘린 나무 길이가 M 이상이면 절단기 높이를 더 높여도 되는지 확인
  - M보다 작으면 절단기 높이를 낮춰야 하므로, 높이를 기준으로 이분탐색 진행
- 고려한 시간복잡도:
  - 나무 수 N <= 1,000,000 이므로 높이를 1씩 바꾸며 전부 확인하는 방식은 불가능
  - 각 높이에 대해 잘린 나무 길이 합을 구하는 데 O(N) 높이를 이분탐색하면 O(log H)이므로
  - 전체 시간복잡도는 O(N log H) = 1,000,000 \* log 1,000,000,000(= 30)
- 막힌 부분 / 해결 방법: X

### ⭐️ 문제에서 주로 사용한 알고리즘

- 이분탐색

### 🧠 대략적인 코드 설명

1. 나무 높이 배열 정렬
2. 이분탐색 중간값 mid를 현재 절단기 높이로 설정
3. 각 나무에 대해 tree - mid가 양수인 만큼만 더해 잘린 나무 총합 계산
   - 잘린 나무 길이 합이 M 이상이면 현재 높이도 가능하므로 정답 후보 갱신, 더 높은 높이도 가능한지 보기 위해 left = mid + 1
   - 잘린 나무 길이 합이 M보다 작으면 현재 높이가 너무 높으므로 right = mid - 1
4. 탐색이 끝난 뒤 가능한 최대 절단기 높이 출력
