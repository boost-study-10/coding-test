### 📖 풀이한 문제

- 백준 1253 - 좋다
- 문제 링크: https://www.acmicpc.net/problem/1253
- 난이도: 골드4
- 걸린 시간: 37분 / 2트 / 반례 참고

### 🧩 풀이 설계 / 막힌 부분

- 문제: N개 수 중에서 다른 수 두개의 합으로 나타낼 수 있는 수의 개수 구하기
- 처음 접근 아이디어:
  - 수열의 순서는 중요하지 않으므로 먼저 정렬
  - 각 수를 하나씩 기준으로 잡고
  - 나머지 수들 중 두 수의 합으로 만들 수 있는지 투포인터로 확인
- 고려한 시간복잡도:
  - N = 2,000 이므로 모든 수에 대해 두 수를 탐색하는 O(N^3) 방식은 불가능
  - 정렬 후 각 수마다 투포인터를 수행하면 O(N \* N)
- 막힌 부분 / 해결 방법:
  - 현재 찾고 있는 수의 인덱스를 투포인터가 가리키는 경우를 어떻게 처리할지
    - 처음에는 합을 먼저 계산한 뒤 index와 겹치면 포인터를 이동하는 방식으로 구현
      하지만, 자기 자신을 포함한 합을 먼저 계산하게 되어 잘못된 비교가 발생
    - 합을 계산하기 전에 index left, right 인지 먼저 검사하고, 현재 수를 제외한 뒤에만 합을 비교하도록 처리

### ⭐️ 문제에서 주로 사용한 알고리즘

- 투포인터

### 🧠 대략적인 코드 설명

1. 입력받은 수열을 오름차순으로 정렬
2. 배열의 각 원소를 하나씩 좋은 수인지 확인
3. left = 0, right = n - 1로 두 포인터를 두고 탐색 시작
   1. 현재 찾고 있는 수의 인덱스를 포인터가 가리키면 넘어가기
   2. 두 수의 합 arr[left] + arr[right]에 대해
      - 좋은 수보다 크면 right 감소
      - 좋은 수보다 작으면 left 증가
      - 같으면 정답 개수 추가하고 탐색 종료
4. 모든 수에 대해 반복한 뒤 정답 출력
