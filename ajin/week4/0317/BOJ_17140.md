### 📖 풀이한 문제

- 백준 17140 - 이차원 배열과 연산
- 문제 링크: https://www.acmicpc.net/problem/17140
- 난이도: 골드4
- 걸린 시간: 74분 / 3트 / 오래 걸리면 안되는 문제......

### 🧩 풀이 설계 / 막힌 부분

- 처음 접근 아이디어:
  - 배열 R, C 크기 비교 -> 연산 선택 -> 연산 수행 후 반환한 배열로 업데이트 -> 종료 조건 확인
  - 연산에서 숫자 개수를 집계하므로 Map을 사용해 계산
- 고려한 시간복잡도:
  - r = 100, c = 100, 최대 100초일 때,
  - 연산 작업은 O(R _ C), 각 행 정렬하면 (100 _ log100) 최악의 경우 O(100 _ 100 _ 100 log100)
- 막힌 부분 / 해결 방법
  - 배열 돌리고.. 복사하고 하는 부분.... 기억력 이슈..
  - C연산은 배열 행렬 바꾸고 R연산 진행 후 다시 전치한 값과 동일

### ⭐️ 문제에서 주로 사용한 알고리즘

- 그냥 구현

### 🧠 대략적인 코드 설명

1. 현재 배열의 목표 값이 일치하면 종료, 최대 100회 반복
2. 배열의 행과 열의 개수 비교해 행 개수가 크거나 같으면 R연산, 열 개수가 크면 C연산 수행
3. R연산(각 행을 순회하면서 0을 제외한 횟수 세기)
   1. 숫자와 횟수를 Map으로 저장하고 이를 배열로 변환해
   2. 횟수 오름차순, 숫자 오름차순으로 정렬
   3. 새로운 배열에 행 결과 값 저장
   4. 가장 긴 행 길이 기준으로 나머지 행은 0으로 채우기
4. C연산은 행열을 전치한 후 R연산을 수행하고 다시 전치해서 반환

#### C연산

1. 초기 배열

   ```
   1 2 1
   2 1 3
   3 3 3
   ```

2. 전치

   ```
   1 2 3
   2 1 3
   1 3 3
   ```

3. R연산 수행

   ```
   1 1 2 1 3 1
   1 1 2 1 3 1
   1 1 3 2 0 0
   ```

4. 전치
   ```
   1 1 1
   1 1 1
   2 2 3
   1 1 2
   3 3 0
   1 1 0
   ```
