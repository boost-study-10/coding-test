### 📖 풀이한 문제

- 백준 2003 - 수들의 합 2
- 문제 링크: https://www.acmicpc.net/problem/2003
- 난이도: 실버4
- 걸린 시간: 17분 / 1트

### 🧩 풀이 설계 / 막힌 부분

- 문제: 연속된 부분 수열의 합이 M이 되는 경우의 수 구하기
- 처음 접근 아이디어:
  - 누적합 먼저 저장하기
  - left, right 두 포인터로 구간 움직이기
  - 현재 구간 합이 M보다 작으면 right 증가
  - 현재 구간 합이 M보다 크면 left 감소
  - 현재 구간 합이 M과 같으면 정답 개수 증가
- 고려한 시간복잡도:
  - N = 10,000 이므로 브루트포스 시 O(N^2) 애매..
  - 투포인터 사용하면 O(N), 누적합 배열 만들 때 O(N) -> 전체 시간복잡도 O(N)
- 막힌 부분 / 해결 방법:
  - 사실 누적합 없이도 풀 수 있는 투 포인터 문제인데,
  - 처음에는 구간 합 문제라고 보고 누적합을 꼭 써야 하나 고민하는 과정에서 시간이 오래 걸림

### ⭐️ 문제에서 주로 사용한 알고리즘

- 투포인터
  - 수열의 원소가 모두 자연수이므로
    구간을 오른쪽으로 늘리면 합이 커지고 왼쪽으로 줄이면 합이 작아짐

### 🧠 대략적인 코드 설명

1. 수열의 누적합 배열을 만들기
2. `left`와 `right` 두 포인터로 순회하기
   - 현재 구간 합: `sum_arr[right] - sum_arr[left]`
   - 구간 합 === `M`이면 정답 개수를 1 증가시키고, 더 긴 구간을 보기 위해 `right` 이동
   - 구간 합 > `M`이면 합을 줄여야 하므로 `left` 이동
   - 구간 합 < `M`이면 합을 늘려야 하므로 `right` 이동
3. `right`가 끝까지 가면 종료하고 정답 출력

#### 누적합 배열 없이 풀기

- 현재 구간 합 `sum`을 변수 하나로 직접 관리하고
- `right`를 늘릴 때는 `sum += arr[right]`
- `left`를 줄일 때는 `sum -= arr[left]`
