### 🧩 풀이 설계 / 코드 설명

#### 시간 복잡도

1. 부르트포스 → 가능
   - 최악의 경우: n = 50, m = 50 일 때, 보드판의 모든 칸(8\*8)을 다시 칠하기
   - W/B로 시작하는 체스판: 2가지 경우
   - O((n-7)(m-7) × 64 × 2) = 약 **200,000**
2. 생략...

#### 설계

1. 보드를 8\*8로 자르기
2. 체스 보드와 같은 색으로 다시 색칠한 칸 수 구하기
   - 체스 보드는 인접한 칸의 색이 다름
   - W로 시작하는 보드
     - W: 가로/세로 index가 모두 짝수 또는 홀수
     - B: 그 외
   - B로 시작하는 보드
     - B: 가로/세로 index가 모두 짝수 또는 홀수
     - W: 그 외
3. 최소 개수 업데이트

#### 체스판 패턴(홀짝홀짝)

```js
const sign = (i + j) % 2;
if (!sign) {
  // m + n이 짝수
  if (board[row + i][col + j] !== colors[sign]) cnt++;
} else {
  // m + n이 홀수
  if (board[row + i][col + j] !== colors[sign]) cnt++;
}
```

- 체스판은 한 칸 이동할 때마다 색이 바뀜
  - 오른쪽/아래로 한 칸 이동 → 색 반전
  - i번 아래 + j번 오른쪽 = 총 i+j번 이동이고,
    이동을 홀수 번 하면 색이 바뀌고, 이동을 짝수 번 하면 색이 유지됨
- 8×8 체스판에서 (i, j)의 색은 (i + j) % 2 값으로 결정
  - (i + j) % 2 === 0: 시작하는 색과 같은 색
    (i + j) % 2 === 1: 시작하는 색과 반대 색
