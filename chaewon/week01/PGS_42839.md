### 🧩 풀이 설계 / 막힌 부분

#### 문제 분석

- numbers는 길이 1 이상 7 이하인 문자열
- numbers는 0~9까지 숫자만으로 이루어져 있음
- 만약 모든 자리가 다른 숫자라면, 최악의 경우 7! -> 최대 14000 정도의 경우의 수 가능
- 앞의 선택이 뒤에 영향을 주지만, 최적화되는 구조가 아님 (중간에 멈출 수 없음)
- 중복 제거 필요(Set)
- 백트래킹(완전탐색)이 필요함

#### 복잡도 계산

1. 순열 조합 생성 O(n! × n)
2. 소수 판별: O(k√m)
3. 최종: O(n! × √(10ⁿ))
4. 공간 복잡도 O(n!)

#### 설계

1. 주어진 number로 만들 수 있는 순열 조합을 모두 생성하고 Set에 저장한다.
2. `sqrt()`를 사용하여, 해당 숫자의 루트 값까지 나누어지는 수가 있는지 검증한다.
3. `result`를 증가시켜 반환한다.

주의할 점: '종이 조각을 붙여서'
처음엔 단순히 Set()에 각 조합을 모두 추가하려고 했는데, 한 숫자는 한 번밖에 쓰지 못한다는 것을 간과함.

### ⭐️ 문제에서 주로 사용한 알고리즘

- 완전탐색 + Set + dfs

### 🧠 대략적인 코드 설명

```js
function solution(numbers) {
    let result = 0;
    const set = new Set();
    const visited = new Array(numbers.length).fill(false);
    
    // 순열 생성
    function dfs(current) {
        if (current.length > 0) set.add(+current);
        
        for (let i = 0; i < numbers.length; i++) {
            if (visited[i]) continue;
            
            visited[i] = true;
            dfs(current + numbers[i]);
            visited[i] = false;
        }
    }
    
    dfs('');
    
    // 소수 판별
    set.forEach((e) => {
        if (e < 2) return;
        
        for (let i= 2; i <= Math.sqrt(e); i++){
            if (e % i === 0) return;
        }
        
        result++;
    })
    
    return result;
}
```
