### 🧩 풀이 설계 / 막힌 부분

#### 문제 분석

- 전체 학생의 수는 2명 이상 30명 이하
- 체육복을 도난당한 학생의 수, 여벌의 체육복을 가져온 학생의 수는 1명 이상 n명 이하이고 중복되는 번호는 없음
- 최대한 많은 학생이 수업을 듣는 게 목표
- 학생 i이 체육복이 없을 때, 빌릴 수 있는 선택지가 앞 번호와 뒷 번호가 있다면 누구에게 먼저 빌리느냐
- 앞 번호부터 차례대로 확인하며 빌릴 수 있다면 즉시 빌린다
- i-1번 학생은 오직 $i$번 학생에게만 도움을 줄 수 있는 마지막 기회일 가능성

#### 복잡도 계산

1. lost 배열을 훑으며 Set 만들기 (L번)
2. reserve 배열을 훑으며 Set 만들기 (R번)
3. 중복 제거 (여벌 있는데 잃어버린 애들 찾기) (L번)
4. 체육복 빌려주기: reserveSet에 남은 애들 (R번)

-> $O(L + R)$

#### 설계

- count = n으로 변수 생성
- lost(i), reserve(j) 배열 정렬한 후 2중 for 문
- i를 해결할 수 있는 j가 없다면 count 감소
- 최종 count 값 도출
-> $O(L \times R)$

-> 다시 set으로 수정함
-> 존재 여부를 탐색

### ⭐️ 문제에서 주로 사용한 알고리즘

Set, Greedy

### 🧠 대략적인 코드 설명

```js
function solution(n, lost, reserve) {
    const lostSet = new Set(lost);
    const reserveSet = new Set(reserve);

    // 중복 삭제
    for (let e of lostSet){
        if(reserveSet.has(e)){
            lostSet.delete(e);
            reserveSet.delete(e);
        }
    }
    
    for (let e of reserveSet) {
        if (lostSet.has(e-1)){
            lostSet.delete(e-1)
        } else if (lostSet.has(e+1)){
            lostSet.delete(e+1);
        }
    }
    
    return n - lostSet.size;
}
```
