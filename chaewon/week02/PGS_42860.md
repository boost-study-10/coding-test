#### ai와 함께 설계...

이동하는 경우의 수를 살펴보자
1. 직진형: 0 → 1 → 2 → 3 ... → n-1 (한 방향으로 쭉): `name.length - 1`
2. 앞 우선형 (오른쪽 찍고 턴): 0 → 앞쪽 목표 → 0 → 뒤쪽 목표: `i * 2 + (name.length - 다음수정지점)`
3. 뒤 우선형 (왼쪽 찍고 턴): 0 → 뒤쪽 목표 → 0 → 앞쪽 목표: `(name.length - 다음수정지점) * 2 + i`

- 여기서 `A`가 등장함
- 1번 기준으로 A 뭉텅이를 그대로 통과한다면, A 뭉텅이의 길이만큼 추가해야 함
- 이럴 땐, A를 통과하기 VS 다시 A 반대편으로 돌아가서 뒤로 넘어가기를 고민해야 함
- 판단 기준: A를 통과하는 값이 뒤로 넘어가서 되돌아가는 비용보다 작아지면 돌아감을 선택함

### ⭐️ 문제에서 주로 사용한 알고리즘

greedy + 완탐

-> 현재 위치를 기준으로 여기서 턴을 하는 게 최선인가를 매번 판단함

### 🧠 대략적인 코드 설명

```js
const solution = (name) => {
    let result = 0;
    
    // 1. 우선, a~알파벳까지 상하이동 횟수를 모두 구함
    // A~알파벳 VS A-Z-뒤에서 알파벳까지의 경우 중 최소값으로 합산
    const spellMove = [...name].reduce((acc, cur) => acc + Math.min(cur.charCodeAt(0) - 'A'.charCodeAt(0), 'Z'.charCodeAt(0) - cur.charCodeAt(0) + 1), 0);
    
    // 2. 좌우이동 최소 횟수를 구함
    // 1) 직진형
    let cursorMove = name.length - 1;
    for(let i = 0; i < name.length -1; i++) {
        let next = i + 1;
        
        while(next < name.length && name[next] === 'A'){
            next++;
        }
        
        // 2) 앞 우선
        const frontSideMove = i * 2 + name.length - next;
        // 3) 뒤 우선
        const backSideMove = (name.length - next) * 2 + i;
        
        cursorMove = Math.min(cursorMove, frontSideMove, backSideMove);
    }
    return spellMove + cursorMove;
}
```
