## Coding Test Solving Guide

### 문제를 받자마자 확인할 항목

* 시간 제한(Time Limit): 1초 / 2초 / 5초
* 입력 크기 N (최대값)
* 테스트 케이스 수 T
* 입력이 단일 입력인지 / 다중 입력인지
* 입력이 이미 정렬되어 있는지 여부

</br>

### 복잡도

입력 크기 기준으로 알고리즘의 성능을 평가하는 척도

* **시간 복잡도(Time Complexity)**: 입력 크기에 따른 실행 시간 증가량
* **공간 복잡도(Space Complexity)**: 입력 크기에 따른 메모리 사용량

#### 시간 복잡도 개요

| 시간 복잡도     | 의미                   | 코딩 테스트 체감   |
| ---------- | -------------------- | ----------- |
| O(1)       | 상수 시간 (Constant)     | 매우 빠름       |
| O(log N)   | 로그 시간 (Logarithmic)  | 빠름          |
| O(N)       | 선형 시간 (Linear)       | 안전          |
| O(N log N) | 로그 선형 (Linearithmic) | 자주 등장       |
| O(N²)      | 이차 시간 (Quadratic)    | N이 작을 때만 가능 |
| O(N³)      | 삼차 시간 (Cubic)        | 거의 불가       |
| O(2ⁿ)      | 지수 시간 (Exponential)  | 코딩 테스트 불가   |

<br/>

### 3. N 크기별 알고리즘 컷라인

일반적인 제한 시간 **1~5초 기준**, 입력 크기 N에 따라 허용 가능한 알고리즘 범위

| N 범위               | 허용 시간 복잡도    | 대표 알고리즘                                 |
| ------------------ | ------------ | --------------------------------------- |
| **N ≤ 20**         | O(2ⁿ), O(N!) | 완전 탐색(Brute Force), 백트래킹(Backtracking)  |
| **N ≤ 500**        | O(N³)        | 플로이드–워셜(Floyd–Warshall)                 |
| **N ≤ 2,000**      | O(N²)        | 이중 반복문, 2차원 배열 순회                       |
| **N ≤ 100,000**    | O(N log N)   | 정렬(Sort), 이분 탐색(Binary Search), 힙(Heap) |
| **N ≤ 1,000,000**  | O(N)         | 투 포인터(Two Pointer), 슬라이딩 윈도우, 누적 합      |
| **N > 10,000,000** | O(log N)     | 이분 탐색, 매개 변수 탐색                         |

<br/>

### 문제 유형 판단 기준

#### 최단 거리 / 경로 문제

* **가중치 없는 그래프 / 격자** → 너비 우선 탐색(BFS, Breadth-First Search): 내 주변부터 한 칸씩 넓히며 탐색
* **가중치가 있는 그래프 (양수)** → 다익스트라(Dijkstra): 통행료 싼 길부터 먼저 가보기
* **음수 가중치 포함** → 벨만–포드(Bellman–Ford): 보너스 길이 있을 때, 모든 간선 반복 확인
* **모든 정점 간 최단 거리** → 플로이드–워셜(Floyd–Warshall): 3중 루프로 전체 표 기록

#### 연속된 구간 / 배열 조건 문제

* **정렬된 배열 + 양 끝 이동** → 투 포인터(Two Pointer): L, R 포인터 조작
* **고정된 구간 크기 유지** → 슬라이딩 윈도우(Sliding Window): 구간 유지하며 이동
* **구간 합을 여러 번 계산** → 누적 합(Prefix Sum): 이전 합 활용

#### 선택의 영향 범위 판단

* **현재 선택이 이후 결과에 영향 없음** → 그리디(Greedy): 동전 거스름돈처럼 앞뒤 안 잴 때
* **이전 선택이 이후 상태에 영향** → 동적 계획법(DP, Dynamic Programming): 기록해두고 재사용, 점화식 사용

#### 중복 / 빈도 / 존재 여부

* 데이터 존재 여부/빈도 → **Hash Map / Set** ($O(1)$ 접근)

<br/>

### 알고리즘별 접근 전략

#### 완전 탐색 (Brute Force)

* **OK**: $N \le 2,000$, 중첩 루프 2개 이하, 단순 연산, break/continue 가능 시
* **CUT**: $N \ge 10,000$, 3중 루프 필요, 모든 조합 요구 시 → **즉시 구조 변경**

#### 백트래킹 (Backtracking)

* 탐색 중 조건 불만족 시 즉시 되돌아감
* DFS: 모든 경로 탐색 목적
* 백트래킹: **정답 탐색 + 불필요한 경로 차단**
* 예시: 합이 특정 값 초과 시 하위 탐색 중단

#### 그리디 (Greedy)

* 정렬 후 순차 선택 가능한지 확인
* 현재 선택이 미래 선택에 악영향 없는지 반례 점검
* 반례가 명확하면 즉시 보류

#### 그래프 탐색 (Graph Traversal)

* **공통** 연결, 이동, 도달 가능 여부
* **BFS** 최소 거리 / 최소 횟수 보장, 큐(Queue) 사용
* **DFS** 모든 경로 탐색, 스택(Stack) / 재귀 사용

#### 최적화 패턴 (O(N²) → O(N))

* **해시(Hash)** 중첩 반복 중 하나가 “이미 본 값” 탐색일 때
* **투 포인터(Two Pointer)** 정렬된 배열에서 합 / 길이 조건 탐색
* **누적 합(Prefix Sum)** 구간 합을 반복 계산할 때

<br/>

### 요약 키워드

* 모든 경우를 다 본다 → 완전 탐색(Brute Force)
* 가다가 막히면 되돌아간다 → 백트래킹(Backtracking)
* 가까운 순서로 확장 → BFS
* 비용이 싼 순서로 선택 → 다익스트라(Dijkstra)
* 구간 / 합 / 범위 → 투 포인터 / 누적 합
* 지금이 제일 중요 → 그리디(Greedy)
* 기록해두고 재사용 → 동적 계획법(DP)
* 최솟값의 최댓값 → 이분 탐색 / 매개 변수 탐색
* 경우는 많고 N은 큼 → DP 또는 그리디
* 사전순 / 가장 빠른 선택 → 그리디 또는 BFS
